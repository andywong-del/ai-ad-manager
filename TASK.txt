========================================================================
 #1 (M1) 结构化 block JSON schema 校验 — 5 个具体任务
========================================================================

背景说明：
  AI 在 SSE 文本里输出 fenced block（```setupcard / ```metrics 等），
  客户端 ChatInterface.jsx:211-261 对每个块 JSON.parse。
  当前 server 零校验，AI 一旦吐出尾逗号 / 缺引号的 JSON，
  客户端 catch 分支就把整段 fallback 成原始代码块 —— 用户
  看到光秃秃的 JSON，而不是卡片 / 按钮。

目标：
  在 server SSE 管道里拦住坏 JSON，保证前端只收到合法块。
